SMT API (enginerpc.com)

A SMT API é um serviço crucial para a indexação de comentários e a gestão de interações sociais relacionadas a tokens na blockchain Hive.

Endpoints de Estado e Configuração

GET /state

Retorna o estado atual do sincronizador de blocos da SMT API.

Parâmetros: Nenhum
bash 
curl https://smt-api.enginerpc.com/state
GET /info

Fornece informações sobre o pool de recompensas de um token específico.

Parâmetros:
  • token (obrigatório): Símbolo do token (ex: BYTE, BEE)
bash 
curl https://smt-api.enginerpc.com/info?token=BYTE
GET /config

Retorna as configurações de um token, como o ID do pool de recompensas.

Parâmetros:
  • token (opcional): Símbolo do token
bash 
curl https://smt-api.enginerpc.com/config?token=BYTE

Endpoints de Feed e Discussões

Estes endpoints permitem a recuperação de diferentes tipos de feeds e discussões, com opções de filtragem e paginação.

GET /get_feed

Retorna o feed de um usuário, incluindo posts e reblogs.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • tag (obrigatório): Nome da conta do usuário
  • limit (opcional, padrão: 20): Número máximo de posts
  • include_reblogs (opcional, padrão: true): Incluir reblogs
bash 
curl "https://smt-api.enginerpc.com/get_feed?limit=20&tag=faireye&token=BYTE&include_reblogs=true"
GET /get_discussions_by_created

Retorna discussões ordenadas pela data de criação.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • limit (opcional, padrão: 20): Número máximo de posts
  • tag (opcional): Filtrar por tag
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_created?limit=200&token=BYTE"
GET /get_discussions_by_trending

Retorna discussões em alta, baseadas em um algoritmo de pontuação.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • limit (opcional, padrão: 20): Número máximo de posts
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_trending?token=BEE"
GET /get_discussions_by_hot

Retorna discussões quentes, baseadas em um algoritmo de pontuação.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • limit (opcional, padrão: 20): Número máximo de posts
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_hot?token=BEE"
GET /get_discussions_by_promoted

Retorna discussões promovidas.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • limit (opcional, padrão: 20): Número máximo de posts
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_promoted?token=BEE"
GET /get_discussions_by_comments

Retorna comentários feitos por um usuário.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • tag (obrigatório): Nome da conta
  • limit (opcional, padrão: 20): Número máximo de posts
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_comments?limit=200&token=BYTE"
GET /get_discussions_by_replies

Retorna respostas recebidas por um usuário.

Parâmetros:
  • token (obrigatório): Símbolo do token
  • tag (obrigatório): Nome da conta
  • limit (opcional, padrão: 20): Número máximo de posts
bash 
curl "https://smt-api.enginerpc.com/get_discussions_by_replies?limit=20&tag=faireye&token=BYTE"

Endpoints de Conta e Conteúdo Específico

GET /@<account>

Retorna o saldo e outros dados da conta para um token específico.

Parâmetros:
  • token (opcional): Símbolo do token
bash 
curl "https://smt-api.enginerpc.com/@sagarkothari88?token=BYTE"
GET /@<account>/<permlink>

Retorna detalhes de um post específico, incluindo votos ativos.

Parâmetros:
  • token (opcional): Símbolo do token
bash 
curl "https://smt-api.enginerpc.com/@sagarkothari88/cc8a12ab?token=BYTE"
GET /get_account_history

Retorna o histórico de transações de uma conta para um token específico.

Parâmetros:
  • account (obrigatório): Nome da conta
  • token (opcional): Símbolo do token
  • limit (opcional, padrão: 100): Número máximo de transações
  • offset (opcional, padrão: 0): Deslocamento para paginação
bash 
curl "https://smt-api.enginerpc.com/get_account_history?account=faireye&token=BYTE&limit=30&offset=0"
GET /get_staked_accounts

Retorna uma lista de contas com tokens staked.

Parâmetros:
  • token (obrigatório): Símbolo do token
bash 
curl "https://smt-api.enginerpc.com/get_staked_accounts?token=BEE"

Hive Engine JSON-RPC API

Esta é a API principal para interagir com os smart contracts do Hive Engine. Ela permite a execução de métodos RPC para consultar o estado da blockchain e dos contratos.

Endpoints Principais

POST /blockchain

Consulta informações sobre os blocos da sidechain.

Métodos RPC:
  • getLatestBlockInfo(): Obtém o último bloco
  • getBlockInfo(blockNumber): Obtém informações de um bloco específico
bash 
curl -X POST https://api.hive-engine.com/rpc/blockchain \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "getLatestBlockInfo",
    "id": 1
  }'
POST /contracts

Permite interagir com os smart contracts do Hive Engine (tokens, mercado, etc).

Métodos RPC:
  • find(contract, table, query): Encontra documentos em uma tabela
  • findOne(contract, table, query): Encontra um único documento
bash 
curl -X POST https://api.hive-engine.com/rpc/contracts \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "find",
    "params": {
      "contract": "tokens",
      "table": "tokens",
      "query": {"symbol": "BEE"}
    },
    "id": 1
  }'

History API (history.hive-engine.com)

A History API fornece um histórico detalhado de transações para contas e tokens específicos.

GET /accountHistory

Obtém o histórico de transações de uma conta para um token específico.

Parâmetros:
  • account (obrigatório): Nome da conta
  • symbol (opcional): Símbolo do token
  • limit (opcional, padrão: 500): Número máximo de transações
  • offset (opcional, padrão: 0): Deslocamento para paginação
  • type (opcional): Tipo de transação (ex: user, contract)
bash 
curl "https://history.hive-engine.com/accountHistory?account=faireye&symbol=BYTE&limit=30&offset=0"
